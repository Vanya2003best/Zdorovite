## 0 · Контекст и жёсткие правила (читать перед ЛЮБЫМ этапом)

### Проект
- **NaZdrow!** — платформа персональных тренеров (Польша). UI-язык — **польский**.
- Стек: **Next.js 16.2.4** (App Router, код в `src/`), React 19, Tailwind v4, Supabase (SSR).
- Supabase проект `cnrgttflzxwcahlbhnsc`; миграции в `supabase/migrations/` (последняя — `031`).
- Dev: `npm run dev` → localhost:3000. Windows-шишка: фоновый dev-сервер не убивается
  штатно — добивать `taskkill /PID <pid> /F`, иначе порт 3000 занят.

### ⚠️ Next.js 16 — НЕ тот Next, что в обучающих данных
Перед нетривиальным кодом читать `node_modules/next/dist/docs/` (см. AGENTS.md).
Breaking changes в API/конвенциях — не писать «по памяти».

### Навигация по коду
`graphify query "<вопрос>"` (граф в `graphify-out/`), после правок — `graphify update .`.

### Бизнес-гвардрейлы (нарушение = брак)
1. **Никаких комиссий с сессий.** Деньги клиент→тренер идут напрямую (BLIK/przelew/gotówka,
   «płatność u trenera»). НЕ добавлять Stripe/P24 в бронирование, НЕ добавлять
   merchant-флоу. Монетизация = подписка тренера + промо. (Фаза 1 запуска — вообще без
   in-platform оплат.)
2. **MVP-скоуп заморожен** (25 экранов). Новые страницы/фичи НЕ добавлять. В частности OUT:
   recurring billing, WhatsApp/SMS, Apple Health, native video, Google Calendar sync
   (только .ics), помиары тела кроме веса, compare trainers, gamification.
3. **`/studio/pages` остаётся за флагом** `ENABLE_PAGES=false` (`src/lib/feature-flags.ts`) —
   не включать, не удалять.
4. **6 шаблонов профиля** (Premium, Cozy, Studio, Cinematic, Luxury, Signature) — все
   остаются; Free = Premium+Cozy, PRO = остальные 4.

### Дизайн-гвардрейлы (утверждены Ваней, не «улучшать»)
- Контейнер **фиксированный `max-w-[1200px]`** — не расширять.
- Типографика фиксированная по токенам дизайна — **НЕ вводить `clamp()`**.
- Единственный wide-screen трюк: `@media (min-width:1500px){ html { zoom: 1.1 } }` в
  `src/app/globals.css` — не трогать, не дублировать другими механизмами.
- Body белый, без градиентных полей; `bg-slate-50` у Top-trainers-секции и Footer — оставить.

### Правила качества (Ваня отвергает «быстро, но тяп-ляп»)
- Каждая фича — **вертикальный слайс целиком**: миграция (если нужна) → server action →
  UI → пустые/ошибочные состояния. Никаких моков, никаких «подключим потом».
- Миграции — идемпотентные; применять к live через существующий процесс (Management API),
  НЕ через `db push` вслепую.
- Блокер (нужны ключи/решение Вани) → СТОП и вопрос, не имитировать работу.

### Верификация каждого этапа (обязательный гейт перед коммитом)
1. `npx tsc --noEmit` → **0 ошибок** (базовая линия чистая).
2. `npm run build` → успешно.
3. Ручной смоук затронутых роутов через dev-сервер (проверить 200 + ключевой контент).
4. `npm run lint` — НЕ добавлять новых ошибок (исторический долг ~190 ошибок чинится
   отдельным этапом 4, не смешивать).
5. Коммит: **только свои файлы** (`git add <пути>`, НИКОГДА `git add -A` — в дереве
   лежат чужие правки AGENTS.md/CLAUDE.md и служебные папки). Push в origin master.

BasicForm, LocationForm, SpecializationsForm, SocialForm, CertificationsEditor, AvatarTile,

услуги/пакеты). Тренер не видит, как правки выглядят на публичном профиле.

**Целевая картина (спека 9.3 из PROJECT_PRIORITIES_AND_PLAN.md):** один экран
`/studio/profile` в стиле OLX-объявления: слева — inline-редактируемые секции, справа —
живой предпросмотр публичного профиля; two-page-флоу исчезает. `EditorClient`
**НЕ переписывать** — полноэкранный «Zaawansowany edytor wyglądu» остаётся доступен
кнопкой из нового редактора (drag-секций и тонкие настройки живут там).

Разбито на 4 слайса — каждый самостоятельный, с приёмкой.

### Слайс 1.1 — Каркас: two-pane редактор с живым предпросмотром
- Новый клиентский каркас на `/studio/profile`: слева колонна секций (~560px), справа
  **живой preview** публичного профиля (масштабированный iframe `/studio/profile/preview`
  ЛИБО прямой рендер компонента шаблона — выбрать по коду; preview-роут уже существует).
- Данные тренера грузятся один раз server-side; правки обновляют preview без перезагрузки
  (optimistic state поверх server actions).
- Секции в порядке публичного профиля: Zdjęcie+Imię+Tagline → Lokalizacja → Specjalizacje →
  O mnie → Usługi/ceny → Pakiety → Galeria → Certyfikaty → Social.
- Существующие формы НЕ выбрасывать: их server actions переиспользуются; UI-обёртки
  заменяются на inline-паттерн (см. 1.2).
- Шапка: статус публикации (Opublikowany/Szkic), кнопка «Podgląd publiczny» (новая
  вкладка `/trainers/[me]`), кнопка «Zaawansowany edytor wyglądu» → `/studio/design`.
- **Приёмка:** /studio/profile рендерит две панели; каждая секция раскрывается; preview
  показывает реальный текущий шаблон тренера; ничего из старого функционала не потеряно
  (все поля старых форм доступны); tsc 0, build ok.

### Слайс 1.2 — Inline-редактирование basics (click-to-edit)
- Паттерн OLX: значение выглядит как текст профиля; клик → инпут по месту; blur/Enter →
  server action save; Escape → откат; спиннер/галка статуса сохранения.
- Покрыть: imię/display name, tagline, «O mnie», lokalizacja (miasto + dzielnica),
  social-ссылки, avatar (клик по фото → upload с кропом, реюз AvatarTile-логики).
- Валидация по месту (пустое имя нельзя, tagline max-длина и т.п.) — сообщения по-польски.
- **Приёмка:** каждое поле редактируется и сохраняется без перезагрузки; невалидное значение
  не сохраняется и показывает причину; preview обновляется ≤1с после сохранения; данные
  реально в БД (проверить перезагрузкой).

### Слайс 1.3 — Галерея: upload → Supabase Storage, reorder, гейт публикации
- Секция Galeria: drag-n-drop/выбор файлов → upload в Supabase Storage (bucket для фото
  профиля; если bucket/колонок нет — миграция с RLS «владелец пишет, публика читает»).
- Сжатие/ограничения: max ~12 фото, формат jpg/png/webp, разумный size-лимит; порядок
  хранится и меняется drag-n-drop.
- **Гейт публикации: минимум 3 фото** — статус «Opublikowany» недоступен, пока фото <3
  (подсказка что добавить). Гейт серверный (не только UI).
- Public-профиль (все 6 шаблонов) читает эту галерею — проверить рендер в каждом шаблоне.
- **Приёмка:** загрузка/удаление/перестановка работают и переживают перезагрузку; при <3
  фото публикация блокируется с понятным сообщением; фото видны на публичном профиле.

### Слайс 1.4 — Usługi/Pakiety inline + бейджи листинга
- Цены услуг и пакетов редактируются click-to-edit прямо в секции (реюз server actions
  из design-редактора); валидация min/max (например 20–1000 zł), save on blur.
- Бейджи карточки каталога (Zweryfikowany/Promocja/Online) и price-line «od X zł» на
  превью соответствуют данным листинга — единая деривация с каталогом (не дублировать
  логику: вынести в общий модуль, если сейчас двойная).
- После этого слайса секции Usługi/Pakiety/ceny в `/studio/design` (EditorClient) должны
  ссылаться на новый редактор ЛИБО остаться, но с одним источником правды — без двух
  разных форм, пишущих одно поле по-разному.
- **Приёмка:** правка цены отражается в preview, на публичном профиле и в карточке
  каталога `/trainers`; невалидная цена отклоняется; нет двух конкурирующих форм.

**Финал этапа 1:** старый двухстраничный флоу исчез из навигации (design остаётся как
«продвинутый» экран), онбординг-ссылки ведут на новый редактор, FIXES/аудит-док не
регрессировал. Полный гейт из §0.
